## 📚 Theoretical Foundations

### Multi-Layer Perceptrons (MLPs)

Multi-layer perceptrons are the foundation of deep learning, consisting of layers of fully connected neurons with non-linear activation functions.

#### Mathematical Formulation

For an MLP with L layers, the forward pass is:

```
h⁽⁰⁾ = x (input features)

For l = 1, 2, ..., L:
  z⁽ˡ⁾ = W⁽ˡ⁾h⁽ˡ⁻¹⁾ + b⁽ˡ⁾
  h⁽ˡ⁾ = σ(z⁽ˡ⁾)  (for hidden layers)
  
Output: ŷ = h⁽ᴸ⁾
```

Where:
- W⁽ˡ⁾ ∈ ℝⁿˡ×ⁿˡ⁻¹ are weight matrices
- b⁽ˡ⁾ ∈ ℝⁿˡ are bias vectors  
- σ(·) is the activation function
- nₗ is the number of neurons in layer l

#### Activation Functions

**ReLU (Rectified Linear Unit)**:
```
ReLU(x) = max(0, x)
```
- Advantages: Computationally efficient, mitigates vanishing gradients
- Disadvantages: Can cause "dying ReLU" problem

**Tanh (Hyperbolic Tangent)**:
```
tanh(x) = (eˣ - e⁻ˣ) / (eˣ + e⁻ˣ)
```
- Range: [-1, 1], zero-centered output
- Better for hidden layers than sigmoid

**ELU (Exponential Linear Unit)**:
```
ELU(x) = { x           if x > 0
         { α(eˣ - 1)   if x ≤ 0
```
- Smooth, non-zero mean, can handle negative values

### Loss Functions

#### Regression (Continuous Targets)

**Mean Squared Error (MSE)**:
```
MSE = (1/N) Σᵢ₌₁ᴺ (yᵢ - ŷᵢ)²
```

**Mean Absolute Error (MAE)**:
```
MAE = (1/N) Σᵢ₌₁ᴺ |yᵢ - ŷᵢ|
```

**Huber Loss** (robust to outliers):
```
Huber(y, ŷ) = { ½(y - ŷ)²           if |y - ŷ| ≤ δ
              { δ|y - ŷ| - ½δ²      otherwise
```

#### Classification (Binary/Categorical)

**Binary Cross-Entropy**:
```
BCE = -(1/N) Σᵢ₌₁ᴺ [yᵢ log(ŷᵢ) + (1-yᵢ) log(1-ŷᵢ)]
```

**Focal Loss** (for imbalanced datasets):
```
FL = -α(1-pₜ)^γ log(pₜ)
```
Where pₜ = p if y=1, else 1-p

### Regularization Techniques

#### Dropout

Randomly zeros elements during training with probability p:
```
h'ᵢ = { 0      with probability p
      { hᵢ/q   with probability 1-p  (where q = 1-p)
```

Benefits:
- Prevents co-adaptation of neurons
- Reduces overfitting
- Approximates ensemble averaging

#### Batch Normalization

Normalizes inputs to each layer:
```
BN(x) = γ * (x - μ_B) / √(σ²_B + ε) + β
```

Where:
- μ_B, σ²_B are batch mean and variance
- γ, β are learnable parameters
- ε is a small constant for numerical stability

Benefits:
- Accelerates training
- Reduces sensitivity to initialization
- Acts as implicit regularization

#### Weight Decay (L2 Regularization)

Adds penalty term to loss function:
```
L_total = L_original + λ Σ ||W||₂²
```

Effect: Encourages smaller weights, preventing overfitting.

### Optimization Algorithms

#### Stochastic Gradient Descent (SGD)

```
θₜ₊₁ = θₜ - η ∇L(θₜ)
```

With momentum:
```
vₜ₊₁ = βvₜ + ∇L(θₜ)
θₜ₊₁ = θₜ - η vₜ₊₁
```

#### Adam (Adaptive Moment Estimation)

```
mₜ = β₁mₜ₋₁ + (1-β₁)∇L(θₜ)     (first moment)
vₜ = β₂vₜ₋₁ + (1-β₂)[∇L(θₜ)]²   (second moment)

m̂ₜ = mₜ/(1-β₁ᵗ)                (bias correction)
v̂ₜ = vₜ/(1-β₂ᵗ)

θₜ₊₁ = θₜ - η m̂ₜ/(√v̂ₜ + ε)
```

Default hyperparameters: β₁=0.9, β₂=0.999, ε=1e-8

### Learning Rate Scheduling

#### Step Decay
```
η(t) = η₀ * γ^⌊t/s⌋
```

#### Exponential Decay
```
η(t) = η₀ * e^(-λt)
```

#### Cosine Annealing
```
η(t) = η_min + ½(η_max - η_min)(1 + cos(πt/T))
```

#### ReduceLROnPlateau
Reduces learning rate when validation metric stops improving.

## 🏭 Semiconductor Manufacturing Applications

### Tabular Data Characteristics

Semiconductor manufacturing generates high-dimensional tabular datasets:

- **Process Parameters**: Temperature, pressure, flow rates, RF power, time
- **Metrology Data**: Critical dimensions, overlay, film thickness, defect counts
- **Sensor Readings**: Real-time equipment health indicators
- **Derived Features**: Ratios, moving averages, control statistics

### Common Use Cases

#### 1. Yield Prediction
- **Input**: Process recipe parameters, tool conditions
- **Output**: Continuous yield percentage (0-100%)
- **Challenge**: Non-linear relationships, tool-to-tool variation

#### 2. Pass/Fail Classification  
- **Input**: In-line test measurements, process parameters
- **Output**: Binary pass/fail prediction
- **Challenge**: Severe class imbalance (failures <1-5%)

#### 3. Parametric Prediction
- **Input**: Process conditions, incoming material properties
- **Output**: Final electrical parameters (Vt, Ioff, frequency)
- **Challenge**: Complex physics, measurement noise

#### 4. Equipment Health Monitoring
- **Input**: Sensor time series, maintenance history
- **Output**: Tool state classification (normal/drift/fault)
- **Challenge**: Concept drift, seasonal patterns

#### Training Loop Structure

```python
def train_epoch(model, dataloader, optimizer, criterion):
    model.train()
    total_loss = 0
    
    for batch_X, batch_y in dataloader:
        optimizer.zero_grad()
        outputs = model(batch_X)
        loss = criterion(outputs, batch_y)
        loss.backward()
        
        # Gradient clipping for stability
        torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
        
        optimizer.step()
        total_loss += loss.item()
    
    return total_loss / len(dataloader)

def validate_epoch(model, dataloader, criterion):
    model.eval()
    total_loss = 0
    
    with torch.no_grad():
        for batch_X, batch_y in dataloader:
            outputs = model(batch_X)
            loss = criterion(outputs, batch_y)
            total_loss += loss.item()
    
    return total_loss / len(dataloader)
```

#### Early Stopping Implementation

```python
class EarlyStopping:
    def __init__(self, patience=10, min_delta=0):
        self.patience = patience
        self.min_delta = min_delta
        self.counter = 0
        self.best_loss = float('inf')
        
    def __call__(self, val_loss):
        if val_loss < self.best_loss - self.min_delta:
            self.best_loss = val_loss
            self.counter = 0
        else:
            self.counter += 1
            
        return self.counter >= self.patience
```

## 📊 Manufacturing-Specific Metrics

### Prediction Within Specification (PWS)

```python
def prediction_within_spec(y_true, y_pred, tolerance=1.0):
    """
    Percentage of predictions within specification limits.
    Critical for manufacturing where small deviations matter.
    """
    within_spec = np.abs(y_true - y_pred) <= tolerance
    return np.mean(within_spec)
```

### Cost-Sensitive Metrics

```python
def manufacturing_cost(y_true, y_pred, 
                      false_positive_cost=10,  # Unnecessary inspection
                      false_negative_cost=1000):  # Missed defect
    """
    Calculate total cost including misclassification penalties.
    """
    y_pred_binary = (y_pred > 0.5).astype(int)
    
    tp = np.sum((y_true == 1) & (y_pred_binary == 1))
    tn = np.sum((y_true == 0) & (y_pred_binary == 0))
    fp = np.sum((y_true == 0) & (y_pred_binary == 1))
    fn = np.sum((y_true == 1) & (y_pred_binary == 0))
    
    total_cost = fp * false_positive_cost + fn * false_negative_cost
    return total_cost
```

### Yield Loss Estimation

```python
def yield_loss_estimate(y_true, y_pred, 
                       tolerance=1.0, 
                       cost_per_unit=100):
    """
    Estimate yield loss based on prediction errors.
    """
    errors = np.abs(y_true - y_pred)
    out_of_spec = errors > tolerance
    excess_error = np.maximum(0, errors - tolerance)
    
    loss_per_unit = excess_error * cost_per_unit
    total_loss = np.sum(loss_per_unit)
    
    return {
        'total_loss': total_loss,
        'avg_loss_per_unit': np.mean(loss_per_unit),
        'out_of_spec_rate': np.mean(out_of_spec)
    }
```

## 🚨 Common Pitfalls and Solutions

### 1. Vanishing/Exploding Gradients

**Symptoms**:
- Loss doesn't decrease
- NaN values in training
- Extremely large gradient norms

**Solutions**:
```python
# Gradient clipping
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

# Better initialization
def init_weights(m):
    if isinstance(m, nn.Linear):
        nn.init.xavier_uniform_(m.weight)
        nn.init.zeros_(m.bias)

model.apply(init_weights)

# Use batch normalization
```

### 2. Overfitting

**Symptoms**:
- Training loss decreases, validation loss increases
- Large gap between train/validation metrics

**Solutions**:
```python
# Increase regularization
model = MLP(dropout_rate=0.3)  # Higher dropout
optimizer = Adam(lr=0.001, weight_decay=1e-4)  # L2 regularization
```

### 3. Poor Convergence

**Symptoms**:
- Loss plateaus early
- Training instability
- Poor final performance

**Solutions**:
```python
# Learning rate scheduling
scheduler = ReduceLROnPlateau(optimizer, factor=0.5, patience=10)

# Better optimization
```

### Activation Analysis

```python
def get_activations(model, x):
    """Extract intermediate activations for analysis."""
    activations = []
    hooks = []
    
    def hook_fn(module, input, output):
        activations.append(output.detach().cpu().numpy())
    
    for layer in model.children():
        if isinstance(layer, nn.Linear):
            hooks.append(layer.register_forward_hook(hook_fn))
    
    model(x)
    
    # Remove hooks
    for hook in hooks:
        hook.remove()
        
    return activations
```

```python
    """Compare current performance to reference baseline."""
    current_metrics = compute_metrics(y_true, y_pred)
    
    alerts = []
```

### A/B Testing Framework

```python
def ab_test_models(model_a, model_b, X_test, y_test):
    """Compare two models statistically."""
    from scipy import stats
    
    pred_a = model_a.predict(X_test)
    pred_b = model_b.predict(X_test)
    
    mae_a = mean_absolute_error(y_test, pred_a)
    mae_b = mean_absolute_error(y_test, pred_b)
    
    # Bootstrap confidence interval for difference
    n_bootstrap = 1000
    differences = []
    
    for _ in range(n_bootstrap):
        indices = np.random.choice(len(y_test), len(y_test), replace=True)
        mae_a_boot = mean_absolute_error(y_test[indices], pred_a[indices])
        mae_b_boot = mean_absolute_error(y_test[indices], pred_b[indices])
        differences.append(mae_a_boot - mae_b_boot)
    
    ci_lower = np.percentile(differences, 2.5)
    ci_upper = np.percentile(differences, 97.5)
    
    return {
        'mae_difference': mae_a - mae_b,
        'ci_lower': ci_lower,
        'ci_upper': ci_upper,
        'significant': not (ci_lower <= 0 <= ci_upper)
    }
```
